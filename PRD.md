# 财务题库 - 会计刷题程序 PRD

## 一、项目概述

### 1.1 产品定位
面向会计从业者、会计专业学生的**移动端刷题工具**，覆盖**初级、中级、高级**三个会计职称考试级别，用户选择目标考试后，题库内容、知识板块、难度范围自动适配。通过智能推题 + 等级晋阶体系，帮助用户高效备考。

### 1.2 目标用户
- 备考初级会计职称的考生
- 备考中级会计职称的考生
- 备考高级会计职称的考生
- 会计专业在校生（日常练习）
- 企业财务人员知识巩固

### 1.3 技术选型
| 项 | 方案 |
|---|---|
| 前端框架 | 原生 HTML/CSS/JS（SPA 单页应用） |
| UI 风格 | 移动端优先，max-width 480px，卡片式布局 |
| 数据存储 | localStorage（离线优先） + Supabase（可选云端同步） |
| 部署方式 | 静态托管（Vercel / GitHub Pages） |
| 题目数据 | JS 内置 + Supabase 远程加载 |

### 2.2 考试选择页（Exam Select）— 首次进入必选

**触发时机**：首次打开应用时显示，后续可在"我的"页面切换。

**页面内容**：
- 标题：选择你的备考目标
- 三个考试等级卡片（单选，点击选中后高亮）
- 每个卡片显示：考试名称、简介、包含的知识板块预览、题量
- 底部确认按钮："确认选择，开始学习"
- 切换提示："选择后可随时在个人中心切换，切换后学习进度独立保存"

| 考试等级 | 卡片显示内容 |
|---------|------------|
| **初级会计职称** | 会计基础 + 经济法基础，适合零基础入门，侧重基本概念和简单核算 |
| **中级会计职称** | 中级会计实务 + 财务管理 + 经济法，适合有基础者进阶，侧重复杂业务处理 |
| **高级会计职称** | 高级会计实务，适合资深财务人员，侧重综合案例分析和管理决策 |

**选择后的影响**：
1. **题库数据切换**：只加载 `exam` 字段匹配的题目，过滤掉不相关内容
2. **知识板块变化**：首页宫格、题库分类只展示该考试对应的板块
3. **测评内容变化**：入学测评只从对应板块中抽题
4. **晋阶目标变化**：不同考试对应不同的晋阶条件
5. **进度独立**：每个考试等级的学习进度、错题本、等级数据独立存储

**数据隔离设计**：
```javascript
// localStorage key 带考试前缀
`kuaiji_${examLevel}_records`     // 如 kuaiji_junior_records
`kuaiji_${examLevel}_wrong_book`
`kuaiji_${examLevel}_user_profile`
`kuaiji_${examLevel}_done_questions`
```

**切换考试的交互**：
- 在"我的"页面点击"切换考试等级"
- 弹出确认弹窗："切换到 XX 考试，当前考试进度已保存，可随时切回"
- 确认后刷新整个应用状态（题库、首页、错题本全部重新加载）
- 不清除历史数据，各考试数据完全独立

## 四、智能学习系统

### 4.1 等级体系

| 等级 | 名称 | 描述 | 难度偏好 |
|------|------|------|---------|
| Lv.1 | 会计小白 | 借贷都分不清？从头开始 | 简单70% / 中等25% / 困难5% |
| Lv.2 | 初窥门径 | 会计要素已经拿捏了 | 简单60% / 中等30% / 困难10% |
| Lv.3 | 小有心得 | 分录报表不在话下 | 简单30% / 中等50% / 困难20% |
| Lv.4 | 财务能手 | 综合题也能从容应对 | 简单10% / 中等40% / 困难50% |
| Lv.5 | 财务达人 | 没有我不会的会计题 | 简单0% / 中等20% / 困难80% |

> 等级体系在三个考试等级中通用，但晋阶目标板块按各考试的板块分别配置。

### 4.2 入学测评（自适应）

- **题量**：5~12 题（自适应终止）
- **算法**：初始 easy，连续答对 2 题升难度，连续答错 2 题降难度
- **提前终止**：连续 3 题 hard 答对 → Lv.5；连续 3 题 easy 答错 → Lv.1
- **评分映射**：得分 ≤30% → Lv.1，≤50% → Lv.2，≤70% → Lv.3，≤90% → Lv.4，>90% → Lv.5
- **板块轮转**：10 个板块依次出题，确保覆盖面

### 4.3 智能推题算法

1. **板块加权轮转**：每个板块维护动态权重，错题多的板块权重升高（最高 ×3），答对的板块权重降低
2. **难度偏好匹配**：根据当前等级的难度配比选题
3. **已做去重**：优先推送未做过的题，剩余不足 10 题时重置
4. **掌握判定**：某板块累计 ≥30 次且正确率 >80% 标记为已掌握，权重降至最低
5. **权重公式**：`权重 = 1.0 + min(错题数 × 0.15, 2.0) - min(答对数 × 0.05, 1.5)`，下限 0.2

| Lv.4→5 | 300 | ≥75% | ≥70% | ≥90% | 近20题对16 | sr_performance_management, sr_comprehensive |

- **挑战规则**：10 题，答对 8 题通过（从当前考试的目标板块中抽题）
- **冷却机制**：失败后 24 小时才能再次挑战
- **通过奖励**：等级升级动画 + 新等级解锁

## 五、数据存储

### 5.1 本地存储（localStorage）

所有 Key 按考试等级隔离，使用 `kuaiji_${examLevel}_` 前缀：

| Key | 内容 |

| `kuaiji_intermediate_*` | 中级（同上结构） |
| `kuaiji_senior_*` | 高级（同上结构） |
